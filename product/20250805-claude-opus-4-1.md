# Claude Opus 4.1 发布

**日期：** 2025年8月5日  
**类型：** 产品发布

## 描述

Claude Opus 4.1 相比 Opus 4 实现了渐进但显著的改进，在 SWE-bench 上达到 74.5%，并引入了增强的多文件重构能力。该模型支持 200K 上下文和 32K 输出 tokens，实现更大的代码库编辑和更长的生成内容。

## 技术细节

- **SWE-bench：** **74.5%**（从 Opus 4 的 72.5% 提升）
- **多文件重构：** 改进了跨多个文件进行协调更改的能力——对实际代码库至关重要
- **上下文：** 200K tokens
- **输出：** 32K token 输出限制——支持更长生成、更大代码块
- **编程：** 软件工程任务最佳水平
- **API：** 完整 API 可用

## 意义与影响

- **编程基准：** 74.5% SWE-bench 持续领先
- **实用编程：** 多文件重构满足实际开发者工作流需求
- **输出长度：** 32K 输出支持更实质性的生成
- **迭代：** 展示了 Anthropic 能力改进的快速发布节奏

# Claude Opus 4.1 Launch

**Date:** August 5, 2025  
**Type:** Product Launch

## Description

Claude Opus 4.1 delivered incremental but significant improvements over Opus 4, achieving 74.5% on SWE-bench and introducing enhanced multi-file refactoring capabilities. The model supported 200K context and 32K output tokens, enabling larger codebase edits and longer generated content.

## Technical Details

- **SWE-bench:** **74.5%** (up from 72.5% in Opus 4)
- **Multi-File Refactoring:** Improved ability to make coordinated changes across multiple files—critical for real-world codebases
- **Context:** 200K tokens
- **Output:** 32K token output limit—enabled longer generations, larger code blocks
- **Coding:** Best-in-class for software engineering tasks
- **API:** Full API availability

## Significance & Impact

- **Coding Benchmark:** 74.5% SWE-bench continued leadership
- **Practical Coding:** Multi-file refactoring addressed real developer workflow needs
- **Output Length:** 32K output enabled more substantial generations
- **Iteration:** Demonstrated Anthropic's rapid release cadence for capability improvements
