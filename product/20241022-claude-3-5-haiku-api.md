# Claude 3.5 Haiku API 发布

**日期：** 2024年10月22日  
**类型：** 产品发布

## 描述

Claude 3.5 Haiku 将 3.5 Sonnet 架构带到 Haiku 层级，以最经济的定价提供了令人惊讶的基准性能——在某些任务上超越 Claude 3 Opus。每百万输入 tokens 0.80 美元，为高容量应用提供了卓越价值。

## 技术细节

- **性能：** 在部分基准上超越 Claude 3 Opus——对于"效率"层级来说很不寻常
- **定价：** **每百万输入 tokens 0.80 美元**——最经济的 Claude 模型
- **速度：** 保持 Haiku 的快速推理
- **上下文：** 200K tokens
- **视觉：** 完整视觉能力
- **用例：** 高容量处理、成本敏感型应用、实时系统

## 意义与影响

- **价值突破：** 在某些任务上以 Haiku 定价提供 Opus 级性能
- **成本降低：** 实现了此前不经济的新用例
- **层级模糊：** 挑战了能力严格遵循层级的假设
- **采用：** 吸引了高容量客户（支持、审核、提取）
- **竞争压力：** 迫使竞争对手改进效率层级产品

# Claude 3.5 Haiku API Launch

**Date:** October 22, 2024  
**Type:** Product Launch

## Description

Claude 3.5 Haiku brought the 3.5 Sonnet architecture to the Haiku tier, delivering surprising benchmark performance—surpassing Claude 3 Opus on some tasks—at the most affordable pricing. At $0.80 per million input tokens, it offered exceptional value for high-volume applications.

## Technical Details

- **Performance:** Surpassed Claude 3 Opus on select benchmarks—unusual for "efficiency" tier
- **Pricing:** **$0.80 per million input tokens**—most affordable Claude model
- **Speed:** Maintained Haiku's fast inference
- **Context:** 200K tokens
- **Vision:** Full vision capability
- **Use Cases:** High-volume processing, cost-sensitive applications, real-time systems

## Significance & Impact

- **Value Breakthrough:** Opus-level performance at Haiku pricing on some tasks
- **Cost Reduction:** Enabled new use cases previously uneconomical
- **Tier Blurring:** Challenged assumption that capability strictly follows tier
- **Adoption:** Attracted high-volume customers (support, moderation, extraction)
- **Competitive Pressure:** Forced competitors to improve efficiency-tier offerings
