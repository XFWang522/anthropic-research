# Claude 3 Haiku 发布

**日期：** 2024年3月13日  
**类型：** 产品发布

## 描述

Claude 3 Haiku 被定位为 Claude 3 系列中最快、最经济的模型，针对高吞吐量应用优化，其中延迟和成本比最大能力更重要。该模型实现了每秒 21,000 tokens 的吞吐量，使其适用于实时应用。

## 技术细节

- **速度：** **每秒 21,000 tokens**——Claude 3 系列中最快
- **定价：** 输入/输出 tokens 最经济的层级
- **上下文：** 200K tokens（与 Opus 和 Sonnet 相同）
- **视觉：** 包含完整视觉能力
- **用例：**
  - 客户支持自动化
  - 内容审核
  - 高容量数据提取
  - 实时助手
  - 成本敏感型应用

- **延迟：** 典型提示的亚秒级响应时间

## 意义与影响

- **市场覆盖：** 满足成本敏感和延迟敏感的细分市场
- **吞吐量基准：** 21K tokens/秒为推理速度设定了高标准
- **采用：** 降低了开发者和初创公司使用 Claude 的门槛
- **互补：** 实现分层架构——Haiku 处理简单任务，Opus 处理复杂任务

# Claude 3 Haiku Launch

**Date:** March 13, 2024  
**Type:** Product Launch

## Description

Claude 3 Haiku was positioned as the fastest and most affordable model in the Claude 3 family, optimized for high-throughput applications where latency and cost matter more than maximum capability. The model achieved 21,000 tokens per second throughput, making it suitable for real-time applications.

## Technical Details

- **Speed:** **21,000 tokens/second**—fastest in the Claude 3 family
- **Pricing:** Most affordable tier for input/output tokens
- **Context:** 200K tokens (same as Opus and Sonnet)
- **Vision:** Full vision capability included
- **Use Cases:**
  - Customer support automation
  - Content moderation
  - High-volume data extraction
  - Real-time assistants
  - Cost-sensitive applications

- **Latency:** Sub-second response times for typical prompts

## Significance & Impact

- **Market Coverage:** Addressed cost-sensitive and latency-sensitive segments
- **Throughput Benchmark:** 21K tokens/sec set high bar for inference speed
- **Adoption:** Lower barrier for developers and startups to use Claude
- **Complement:** Enabled tiered architecture—Haiku for simple tasks, Opus for complex ones
